這是一個非常經典的 「坐標系對齊」 問題。如果角色圖片的 左上角 在屏幕中央，說明 Sprite 的 Offset（偏移量）目前是 (0, 0) 或者正值，導致圖片是從中心點往右下方繪製的。

要讓角色的「腳底」或「中心」對齊屏幕中央，建議從以下三個邏輯層面進行檢查和調整：

1. 理解「負向補償」邏輯 (核心原理)
在 Godot 中，當 Centered = false 時，圖片的左上角默認對齊 Node 的 (0, 0)。

如果 PngInfo 告訴你腳底在圖片中的坐標是 (15, 58)。
這意味著腳底距離圖片左邊緣 15 像素，距離頂部 58 像素。
建議： 你必須把圖片向左移動 15 像素，向上移動 58 像素。反映在代碼上，Sprite 的 Offset 必須設置為 (-15, -58)。
檢查點： 請確認你的代碼是否真的執行了「取反」操作。如果日誌顯示 Anchor 是 (15, 58)，而 FinalOffset 也是 (15, 58)，那角色一定會掉到屏幕下方。
2. 區分「素材偏移」與「全局修正」
目前的 BodyOffset 應該承擔兩個不同的職責，建議你在心裡把它們拆開：

素材偏移 (Anchor)： 這是每一幀都在變的，用來抵消圖片裁剪產生的位移。它必須是 (-dx, -dy)。
全局修正 (Global Tweak)： 這是你提到的 y = -20。這通常是因為 Lineage 的原始數據定義的「地平線」與 Godot 攝像機的中心點存在一個固定偏差。
建議： 最終的公式應該是：Offset = (-sprDx + BodyOffset.X, -sprDy + BodyOffset.Y)。
3. 攝像機 (Camera2D) 的檢查
如果 GameEntity 的 Offset 計算正確，但角色依然不在屏幕中心，問題可能出在攝像機上：

檢查點： 檢查 Camera2D 節點的 Offset 屬性。如果攝像機本身有偏移，或者攝像機沒有完全掛載在玩家 Node 的 (0, 0) 位置，也會導致視覺上的偏差。
建議： 確保攝像機的 Position 和 Offset 均為 (0, 0)。
4. 遠程調試驗證 (Remote Scene Tree)
這是一個最直觀的驗證方法：

運行遊戲。
在 Godot 編輯器的場景面板切換到 「Remote (遠程)」 模式。
找到玩家的 MainBody 節點。
手動在編輯器裡修改它的 Offset 數值。
關鍵實驗： 當你發現角色完美居中時，記錄下那個 Offset 數值。然後對比日誌裡的 Anchor 數據。
如果日誌說 Anchor.X = 15，而你手動調成 -15 才對齊，那就證明代碼裡的 「正負號」 依然是反的。
總結建議
目前的故障現象（左上角在中心）強烈暗示：代碼雖然讀取了偏移量，但並沒有正確地將其轉化為「負向補償」。

在 Godot 中，要讓圖片中的某個點 (x, y) 移到 Node 的原點，Sprite.Offset 必須等於 (-x, -y)。請沿著這個數學邏輯重新審視那幾行計算公式。
